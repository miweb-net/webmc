Scripture Editor



Scripture Editor Functions

The Scripture Editor is displayed when the notebook tab icon is pressed or navigated to in any manner (it's the active tab at the top of the figure above). The editor's largest feature is the text area. The image shows a scripture being displayed and a scrollbar on the right side of the window. The scrollbar will appear only when there is additional content that is not in view. In this case, the passage reference (Romans 10:1-3) is not in view and the scrollbar could be used to see it.

Change Bible Version

The bible version dropdown box is at the bottom of the Scripture Editor page. It contains a list of available bible versions that have been installed on the computer. The preferred bible version is displayed when the program is started. The preferred bible version can be set in Preferences. Clicking on the dropdown will display the list of bible versions, as illustrated at right and a different version can be selected by mouse click or keyboard scrolling while the list is displayed.

Scripture Look-up

The editor has additional controls below the large text area. At the lower-left of the editor is an entry widget used for typing a reference, like Hebrews 11:1. To enter a new passage, triple-click (or type <Ctrl + a>) in the entry box. This selects all of the text in it. Then press the <Backspace> key, or just begin typing in your passage reference. The Scripture Editor recognizes many abbreviated book names as well as verse ranges. For example, Heb 11:1-3 will display the first 3 verses of Hebrews the 11th chapter. To make the referenced scripture display, press the <Enter> key in the Reference entry widget, or click on the button to its right. Note that looking up a scripture causes all manual page breaks to be removed, and automatic page breaks are inserted to fit the new text on presentation pages (slides) using the current font.

Copy Text to/from the Editor

The Scripture Editor window is also a general purpose text display area. This means that arbitrary text can be typed in, copied from, and pasted to this editor area. For example, the user can copy text from a text editor or other program using <Ctrl + c>, and then click the mouse in the editor where the copied text should be inserted, and then paste copied text in the editor using <Ctrl + v>. The reverse is true as well. A reference can be looked up (see above), and the scripture text can be selected with the mouse by clicking and dragging the mouse to include the desired text. Then press <Ctrl + c> to copy the text into the clipboard, and then click on another program window (e.g. a text editor) and press <Ctrl + v> to insert the text.

Add/Remove Page Breaks

A page break symbol ¶ is displayed when large fonts (discussed in the next paragraph) are used, or long scripture passages are requested. This is the paging symbol used to tell the presentation software where to divide pages of content. The Media Center automatically repaginates the scripture every time a reference lookup is requested, or a font size change is made using the controls at the far right of the editor. The user can override the automatic page breaks manually by selecting each one and pressing the <Backspace> or <Delete> key to delete it. Page breaks (¶) can be manually inserted into the text. Manual page breaks will not be moved or deleted by the Media Center when automatic repagination occurs. However, when a new scripture is looked up, all manual page breaks are removed when the new scripture is displayed. Adding manual page breaks is done by clicking the mouse in the Scripture Editor window area to set the cursor location, and then pressing the button. Note that when altering the automatic pagination feature, the presentation program will display it in nearly the same font size as it is displayed in the Scripture Editor and will only create new slides (pages) if it encounters the ¶ symbol.

Change Font Size

Font size and page control are tightly integrated in the Media Center. Requesting a reference as described above, the Media Center will use the current font size managed by the spin box to fit the requested verse(s) to a typical TV or projector screen. If the text will not fit on a single presentation page, the Media Center will automatically insert page breaks (indicated by the ¶ symbol) at optimum break points in the text. If the font size appears larger or smaller than desired, the controls on the right side of the displayed value can select a larger (+) or smaller (-) font. When doing so, the text will be repaginated to fit the page based on the new font size.

Generate Scripture Presentation

After the text size and content are satisfactory, it's time to generate the presentation file. Just as with hymn presentations, the Scripture Editor has the same button to generate scripture presentation files for display. After clicking the generate button, the view is automatically switched in the Media Center to the Organizer. The Media Center performs a check to ensure that the scripture presentation file doesn't already exist. If it does, a confirmation dialog is displayed, enabling the user to continue or abort the overwrite operation. If the file exists, and is overwritten, the icon remains in the same location it was in the Organizer. If it is a new file, the scripture icon is appended to the Organizer list.

External Presentation Editing

The Media Center generates a standard presentation file with a .pptx extension, which can be edited outside of the Media Center program to add graphics and other media content. The location of the scripture presentation files is determined in the Preferences. The Media Center Scripture Editor does not read files from the folder location. It only writes files to the folder. So once a file is generated, the Scripture Editor cannot (directly) read its contents. However, the text (only) can be copied manually back in to the Scripture Editor by copying it from the presentation program window and pasting the text into the Scripture Editor as described above. The Organizer can delete the scripture files that have been created in the Editor.
