Hymn Importer


The Importer is a wizard that is accessed from the File > Import menu or by using the keyboard shortcut <Alt + i>. The import wizard is an advanced feature of the Media Center. It is used for importing a hymn into an SQLITE3 database that is compatible with the Media Center. The process of creating an empty Media Center database is currently a manual process, and is beyond the scope of this document. However, adding hymns to an existing database is what this section covers. The wizard progresses through 4 pages:

  1. Title and Authors
  2. Arrangement Order
  3. Image Mapping
  4. Summary and Upload

Title and Authors

The Title and Authors page begins with three fields that describe the hymn: The first is the Database/Book dropdown list. The selection of the database is important because the importer will modify this database and the hymn will then belong to this database. The Hymn Title and Number fields must be filled in to proceed to the next page. The [Next] button will not be enabled until these two fields contain information. When the Title field contains 5 or more characters, the label text Hymn Title: will change from red to black in color. Only a single character is required in the Hymn Number field.

Lyricist And Composer

The Lyricist (author of the hymn text) and Composer (author of the music) fields are optional. If they are not filled in, the wizard will insert Unknown into the database. Similarly the date fields that follow each of the Lyricist and Composer fields are optional and will default to Unknown if not supplied by the user.

Copyright Disclosure

Copyright information is important for presenting the hymns in compliance with federal law. Many hymns are in the public domain because their copyright has expired, or the hymn was never copyrighted. However, it is necessary to site the copyright holder in the Copyright Owner field and enter the date that the copyright was issued in the Date field. There is an uneditable text field that displays the copyright string that will be included in presentation files that are generated from the Media Center program. If the hymn is copyrighted, click the Public Domain check button to reveal the copyright Date, Owner and Sample fields. Notice that the labels next to the Date and Owner fields are red. This means they are required. If the hymn is actually not copyrighted, you can simply click the Public Domain check button again to hide the copyright controls. Otherwise, enter the copyright information in the provided fields and then click the Next button.

Arrangement Order

The Arrangement Order page is where the common order of hymn parts are established. The image at right was captured after some selections were made. When this page is initially displayed, the Part Sequence list at right is empty. The Part Names list always contains these components. The part names are what are picked from to fill the Part Sequence list. This step governs the default arrangement of the song. Careful consideration should be given as to the customary or preferred order for the hymn composition to ensure that it is complete. Here are the steps that were taken to fill the Part Sequence list as it is illustrated above:

  • Double-click Verse 1 in the Part Names list,
  • Double-click the Chorus in the Part Names list,
  • Double-click Verse 2 in the Part Nameslist,
  • Double-click the Chorus (again).
  • Double-click Verse 3 in the Part Names list,
  • Double-click the Chorus (third time).
  • Double-click Optional Ending in the Part Names list,

If a mistake is made while double-clicking the parts, a part (row) can be removed from the Part Sequence list by double-clicking the row.

The [Toggle Language] button below the Part Names list will switch between English and Spanish part names. You should ensure that you have the part names toggled to match the language the hymn text is written in before double-clicking to add part names, because this button only toggles the names in the Part Names list.

Image Mapping

The mapping of hymn part names to image files is another critical step. If a file is incorrectly mapped to a part name, the presentation file generated for this hymn will never appear correctly when displayed on TV screens. If mapped properly, presentation generation of this hymn will display properly every time.

This page contains a [Browse] button and a Images Folder field just below it. When the browse button is pressed, it will open a folder selection dialog. The user is expected to select a folder that contains several PNG image files that represent the hymn parts. The file names are not important, but the PNG format is recommended. Generating the PNG files is beyond the scope of this document. When the folder is selected, the folder browser dialog (not shown) disappears and the Image Files list is filled with all of the PNG files found in the selected folder. DON'T MISS THIS: When the folder is selected, all of the image files that are listed in Image Files list are also DISPLAYED in the Previewer Window of the Media Center application. This is extremely useful for ensuring that the proper number of pages are mapped to the respective hymn parts as described in the next paragraph.

When this page is initially displayed, the Image Files list is empty because the Media Center is waiting for the user to browse to a folder. The Hymn Parts list should contain exactly the list of parts that was created on the previous wizard page in the Part Sequence list, but without redundancy. For example, if you indicated that the Chorus was to be sung multiple times, Media Center remembers the order, but only displays the Chorus once in the Part Names list on this page. The purpose of this page is to match the files in the Image Files list with the respective part name in the Hymn Parts list. The steps below will explain the mapping of image files to hymn parts. But first, notice in the example illustrated above that Verse 1 in red, is already selected. Selection in the Part Names column is indicated by the left-pointing arrow in the box. This means that any image files selected in the Image Files list will be mapped to Verse 1. Note also that if you hover the mouse over a file in the Image Files list, its background will change slightly to match the selected Part color. When you click on the file name in the Image Files column, the color of the row clicked will be changed to match the color of the part selected in the Part Names column. Remember also that the images should be displayed in the Previewer Window of the Media Center window. Now the steps:

  1. Look in the Previewer Window to see which image files comprise the first part (for example, Verse 1). Single-click on each file in the Image Files list that is part of Verse 1.
  2. If an incorrect mapping is made, simply select the Part Name that the file in the Image Files list belongs to, and then click on the image file name. The color of the image file name will match the color of the part name.
  3. Single-click the Chorus in the Hymn Parts list.
  4. Repeat steps 1-4 until all image files are mapped to the correct parts.

The final mapping will be displayed on the following Summary and Upload page. And you can come back to this page using the [Back] button if a mistake was made. If you look at the image above again, you'll see another User Interface element called Select Audio File. This is filled in when the folder browser dialog is closed and a folder is selected. The wizard searches through the folder as it does for image files, and it populates the Select Audio File list with media files with MP3 (perhaps an actual live recording of a group performance of the hymn) and MID (usually generated by music writing software such as MuseScore) extensions. Often there are none or at most one. When it finds one in the folder, it will be selected by default. What this means is that the media file will be mapped to the entire hymn, just as the image files are mapped to the parts. This is how hymn audio samples are associated with the hymns in the database.

Summary and Upload

This is the review page. It contains all of the information provided by the user through the wizard. It should be reviewed carefully before proceding further, because once the [Upload] button is pressed, the database will be filled with the information displayed here, whether it is correct or not. There is NO UNDO button here, but there is a Delete option in the Edit > Delete menu to address such mistakes. But the wizard must be restarted to enter all of the information over again. So the message here is scrutinize the data before pressing [Upload]. When the [Upload] button is pressed, the new hymn will be written to the database, and the Importer Wizard window will close automatically.

At this point, the hymn is in the database and the Media Center will change the notebook view to the search page. And the Search Page hymn book dropdown will be switched to the database into which the hymn was uploaded. By scrolling to the hymn, you can check its appearance, test the audio sample and create a presentation file immediately.
